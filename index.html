What is our ideal?

<icon>name</icon>
* The icon is a single, separable element * It has accessible content * This means we can style and size it

Performance

* Should have minimal footprint * Preferable to load only what you need, in one resource request * No script DOM manipulation

Easy to use

icon { icons: url('default.icons'); }
* Should be easy to use, just link to the resource. * Not to say this is the ideal form, just to say it should be this easy

Raster Icons

picture of a raster icon at large

* You have exact control over every pixel * Well supported, everywhere that uses images 
<i class='icon-book'></i>
i[class^='icon-'] { /* size and position */ }
.icon-book { background: url('book.png'); }

But you're loading multiple resources!

.icon-book { background: url(data:image/png;base64,...); }

Summary

  • Pixel-level control
  • Browser support
  • Fixed size

Icon fonts

Anatomy of an icon font

* A font maps a character or set of characters to their visual representation (glyph). * Generally vector based for clean rescaling * Package for a set of vector artwork * Mapping taken care of by the browser to replace source characters with their vector representation

&#102; f

* A font takes a code point, the numerical representation of the letter on the computer, and maps it to a visual representation * Here we have the code point 102 as an HTML entity translating to 'F' * There are several different encodings that represent the mapping between a number and a letter, in general the one you're going to be working with is probably Unicode * Font formats like OTF support the Unicode code point mapping

&#102;

* So why not just replace the visual representation with our icon (yay!)

Characters to Glyphs

Considerations

  • Accessibility
  • Browser support
  • Fallbacks

ASCII

f

* The simplest mapping is to go from your typical alphabetical characters to your icon * But screen readers will read this as 'f', semantically doesn't make much sense. 
@font-face {
    name: icon-font;
    src: url(icon-font.woff);
}
.icon { font-family: icon-font; }
.icon-cloud:before { content: 'f'; }

Unicode PUA

* Unicode has some areas (code points) set aside that are not and will not be added as part of a language * Fonts and users can use them however they want.

&#xe000; 

@font-face {
    name: icon-font;
    src: url(icon-font.woff);
}
.icon { font-family: icon-font; }
.icon-cloud:before { content: '\e0000'; }
* Everyone else can use the PUA. * This is a block of the PUA in iOS, where the system font uses that area for emoji. * Fallbacks can be confusing. Get another symbol, or just a box.

Ligatures

fi ﬁ

fish ﬁsh

* A more advanced typography feature, ligatures, allows you to combine a sequence of code points into a single glyph (is glyph the right term to use here?) * Goal is to combine marks to flow well together, fl, ff

cloud

* What if, instead, we replace the text 'cloud' with it's icon as a glyph 
.icon {
    font-family: icon-font;
    text-rendering: optimizeLegibility;
    font-feature-settings: 'dlig';
}
.icon-cloud:before { content: 'cloud'; }
